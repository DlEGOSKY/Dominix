# UI_SYSTEM.md — Dominix

## Objetivo visual
Dominix debe sentirse como una mesa táctica premium:
- fondo oscuro
- fichas protagonistas
- score enorme
- feedback limpio
- claridad total

No debe parecer:
- un clon de casino barato
- una app móvil genérica
- una UI sobrecargada de efectos

## Principios visuales
- foco en las fichas
- contraste alto
- información importante arriba
- score y multiplicador muy visibles
- patrones activados claramente destacados
- motion corto y satisfactorio

## Pantallas principales

### Home
- logo
- botón nueva run
- progreso o desbloqueos si existen
- identidad visual fuerte

### Run screen
- mano actual
- cadena en construcción
- score actual
- meta de ronda
- patrones activos
- reliquias activas
- controles mínimos

### Reward screen
- 3 opciones claras
- comparación rápida
- sin exceso de texto

## Reglas visuales
- cero emojis
- cero ruido visual innecesario
- los números deben ser legibles al instante
- las fichas deben sentirse valiosas
- el feedback de una buena jugada debe ser inmediato
